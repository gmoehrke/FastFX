1 /*
2  * FFXBase : Base class for all LED Strip animation effects.
3  *
4  * All static and animated effects should inherit from this class. The 2 key functions to override/implement in descendant
5  * classes are:
6  *
7  * initLeds( CRGB *buffer )
8  * writeNextFrame( CRGB *buffer )
9  *
10  * Most effects can be written by simply overriding one or both of the above methods (see FXCoreEffects.h for examples).
11  *
12  * FFXBase is a descendent of the FlexTimer class which allows for setting the speed/timing of the animation. FXBase also
13  * inherits from FXStateNotifier, which provides a means for sending state notification changes to any descendant of the
14  * FXStateObserver class (defined here as well). FXSegment, which is the "container" for all FXBase objects, is a descendent
15  * of FXStateObserver, so changes to timing or other state info can be pushed back to the "parent" segment.
16  *
17  * Phase/Cycle - Each effect has a phase and a cycle, the cycle is one complete iteration from 1 to numLeds - the phase being
18  * the current index of that iteration.
19  *
20  * VPhase/VCycle - We can also define a "virtual" phase and cycle which allows have a cycle that is larger or smaller than the number
21  * of LEDs on a strip. A larger VCycle is useful if we want to do additional things at either end of a cycle (i.e. where an animation
22  * will scroll off the end of the strip without wrapping around immediately. A smaller VCycle can be used if an animation needs fewer
23  * phases to complete (ex: quickly switching between to colors, jiggling back and forth, etc.) A VCycle is established by calling
24  * setVCycleRange()
25  *
26  * MovementType is defined here, then used by descendant classes to determine the type of movement for a given effect.
27  *
28  * FFXBase also provides several "utility" methods for performing common operations used in animated effects:
29  *
30  * Rotate pixels within a CRGB Buffer, wrapping around at the ends:
31  * rotateBufferForwardwithWrap( ... )
32  * rotateBufferBackwardWithWrap( ... )
33  *
34  * Calculate the "mirror" image position of a given pixel (i.e. same distance from center, on the other side):
35  *
36  * mirror( uint16_t position )
37  *
38  * Several virtual methods are provided, which can be overriden in descendent classes
39  *
40  * onEachSecond(unsigned long) // called for each elapsed second while effect is running (udpate() called within a loop)
41  * onCycleStart()
42  * onCycleEnd()
43  * onVCycleStart()
44  * onVCycleEnd()
45  * onBrightness() // called when brightness is changed (by FXXSegment)
46  * whileFrozen() // called each time update() is called while effect is "Frozen"/paused
47  */
