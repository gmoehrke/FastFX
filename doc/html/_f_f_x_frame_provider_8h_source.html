1 /*
2  * The FFXFrameProvider controls the fetching of frames that get displayed by the FXController. There are several "modes" of operation
3  * that can be utilized:
4  *
5  * Direct mode - (Crossfade=false AND constructed with FASTLed framebuffer passed in the constructor)
6  *
7  * This is the most efficient usage. It uses a single buffer (CRGB[]) that is passed in the constructor.
8  * This is typically the array of CRGB used by fastLED for display and allocates no additional heap.
9  *
10  * Indirect modes - Indirect modes use independent buffer(s) where pixel data is written and maintained. This means that each frame is calculated independently from
11  * what is in the FastLED display buffer. This can be useful for overlaying or mixing effects.
12  *
13  * without crossfade - Uses a single buffer, remains independent of any changes made to the fastLED display buffer.
14  *
15  * with crossfade - Uses 2 buffers to smooth animated effects. A second buffer is used to "smooth" transitions from frame to frame - uses multiple
16  * cycles to blend from the current frame to the next one. The number of blend steps between frames depends on the available time between frames and
17  * the FrameViewController will fill as many steps as it can in that time.
18  */
