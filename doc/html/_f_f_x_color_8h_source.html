6 /* The FFXColor class provides a common interface for dealing with color, color changes, and palettes in effects.
7  *
8  * The FFXBase parent class for all effects has a member named currColor which is available to all descendant classes for
9  * getting color values. The FXColor class supports 4 modes - 2 single color modes and 2 palette modes for effects that use
10  * multiple colors.
11  *
12  * Each effect simply needs to call the getCRGB function to return the currently selected color. This class provides 2 methods
13  * for stepping through palette colors and shifting the palette for "rotating/shifting" palette effects. This class operates
14  * differently depending on the mode selected (use setColorMode() to change the mode) as follows:
15  *
16  * mode singleCRGB - always return the same color, specified by setCRGB
17  * mode singleCHSV - always return the same color (CHSV) as specified by setCHSV
18  * mode palette16 - Returns colors from a 16 entry palette (set with setPalette() )
19  * step() - steps forward 1 entry in the palette. So, if I have a 16 entry palette - each time
20  * step() is called, the next entry will be returned by getCRGB(). Once the last entry is
21  * returned, the next call to step() starts over at the first entry.
22  * shift() - will shift each entry forward n steps where n is the value of shiftDelta
23  * (defaults to 1). Each palette entry returned using a 0-255 index, so as
24  * shift() is called, each entry moves closer to the next entry in the palette. Once
25  * we've shifted forward 16 times, the entire palette has shifted forward 1 step.
26  * setRange() - This will allow the palette16 to work the same with fewer than 16 entries.
27  *
28  * resetStep() - sets the step position back to the first entry in the palette.
29  * resetShift() - Shifts the entire palette back to its original position.
30  *
31  * mode palette256 - Works almost the same way as palette16, however calls to step() only move forward 1 position. Shift will
32  * still shift the palette forward each time it is called as well. This can be used to rotate a palette through a
33  * fixed "window" without needing to track the index ranges. For example - I can display entries 1-100 in a loop and
34  * call shift after each one. This will continually rotate the entire palette through those first 100 entries.
35  *
36  */
